# Gaps

- **Articulated + deformable objects unsupported.** Method hardcodes rigid-body motion (Θobj = R,t,s). Extension needs DINOv2-distilled articulation tracking or DeformGS-style time-varying splats; an obvious lever is to swap SAM3D + MegaPose for a joint articulation-estimator and re-use the same hand-object loss.
- **Quasi-static scene assumption.** Only the manipulated object may move; any scene change (receptacle lifted, liquid poured visibly, second agent) breaks rendering. Dynamic 3DGS (4D Gaussians, Dynamic3DGS) integration is unexplored.
- **Occlusion failure mode.** Full object occlusion collapses the optimizer; Can-on-Plate already shows FoundationPose failing here. A probabilistic / multi-hypothesis object tracker or hand-conditioned object prior (learned from internet ego data) is missing.
- **Retargeting is parallel-jaw-specific.** Thumb+index→EE mapping only. Dexterous (Allegro, Leap), suction, two-finger soft grippers, bimanual — all open. A unified "contact-primitive retargeter" conditioned on gripper URDF would generalize.
- **Pipeline uncertainty is uncalibrated.** Seven foundation models cascade; no per-frame confidence gating, no skip/reject of bad frames before they poison rendered demos. Would benefit from per-stage confidence → weighted loss during policy training, or automatic demo filtering.
- **Rendering-aug mix is load-bearing but under-studied.** Ablation shows 3/5 tasks go to 0/20 without 10x aug. Which aug dimension matters (retexture vs. intrinsics vs. translation) is unknown; likely redundant with simpler 2D aug schemes.
- **No language / task conditioning.** One-policy-per-task, one-object-per-task. A multi-task WARPED conditioned on language prompts (mapped onto Grounding-DINO text anyway) is a low-hanging extension.
- **Contact window detection is threshold-based.** Mask-overlap + motion heuristics miss near-contact hovers and partial grasps. Learning a contact classifier from HOI datasets (HOI4D, ARCTIC) would likely remove a chunk of failure cases.
- **Long-horizon / multi-step manipulation untested.** All five tasks are single-grasp, single-release. Chaining contact segments (pick-place-pick) needs re-entering the scene-init or tracking through multiple contact cycles.
- **Sim-to-real gap measurement is only implicit.** Paper adds Gaussian image noise + aug; no quantification of rendered-vs-real distribution gap (e.g., FID, feature-space distance). Diagnostic metric would let the community measure whether better splats actually translate to better policy.
- **Scene scan amortization not reported.** The "5-8x faster" claim folds the 1-min scan into data-collection time for a fixed scene; scaling to per-demo novel scenes (real mobile manipulation) may erode the advantage. A scan-free variant (use only the demo video for splat init) is an open direction.
- **Co-training incompatibility on fine-motion tasks.** Wipe Brush co-training *hurts* (15 teleop alone beats 15+15 mix). Suggests a demo-quality gate or distributional-alignment loss is needed when mixing WARPED + teleop data.
- **No cross-embodiment transfer.** Rendered wrist view is tied to xArm G1 gripper geometry; swapping to a different EE mid-training would require re-rendering. A gripper-agnostic intermediate representation (e.g., grasp keypoints + relative pose) could unlock embodiment-free pretraining.
- **Bridge to latent WAM pretraining (Being-H0.7) is untouched.** WARPED throws away the latent ego-video representation after extracting geometry; using the same monocular ego stream to *jointly* pretrain a world-action backbone while producing wrist renders is a natural fusion.
